## تغییرات دیتابیس

### Migration جدید: `2026_06_12_000001_modify_form_field_values_for_multi_row.php`

**تغییرات:**
- ❌ حذف `activity_id` (دیگر لازم نیست)
- ✅ اضافه کردن `form_id` (برای ارجاع مستقیم به فرم)
- ✅ اضافه کردن `row_index` (برای ذخیره چند ردیف)
- ✅ Unique constraint جدید: `(form_field_id, form_id, row_index)`

**ساختار قبل:**
```sql
form_field_values:
  - id
  - form_field_id
  - activity_id  ← استفاده نمی‌شد
  - field_value
  - created_by
```

**ساختار بعد:**
```sql
form_field_values:
  - id
  - form_field_id
  - form_id         ← جدید
  - row_index       ← جدید (0, 1, 2, ...)
  - field_value
  - created_by
```

### 2. Controller: `FormController.php`

#### متد `processSheet()` - بازنویسی کامل

**قابلیت‌های جدید:**
- ✅ خواندن **تمام ردیف‌های داده** (از ردیف 7 به بعد)
- ✅ حذف ردیف‌های خالی به صورت خودکار
- ✅ ذخیره اقدام با validation بهتر
- ✅ ذخیره فعالیت با جستجوی کد و عنوان
- ✅ ایجاد `FormFieldValue` برای هر ردیف با `row_index`

**منطق:**
```
1. خواندن کل شیت (ردیف 1 تا آخر)
2. استخراج هدف، برنامه، اقدام، فعالیت (ردیف 1-4)
3. استخراج عناوین فیلدها (ردیف 6)
4. استخراج تمام ردیف‌های داده (ردیف 7+)
5. یافتن ID های مربوطه از دیتابیس
6. ایجاد Form
7. ایجاد FormField ها (یک بار)
8. ایجاد FormFieldValue ها (برای هر ردیف داده)
```

## نحوه استفاده

### 1. اجرای Migration

```bash
php artisan migrate
```

**نکته مهم:** این migration داده‌های قدیمی را حذف می‌کند. اگر داده مهمی دارید، ابتدا backup بگیرید.

### 2. Import فایل Excel با چند ردیف

**مثال فایل Excel:**
```
ردیف 1: هدف 5 : به ثمر رساندن پرونده‌ها
ردیف 2: برنامه 4 : انجام اقدامات حقوق عامه
ردیف 3: اقدام 30201 : نظارت بر محیط زیست
ردیف 4: فعالیت 1 : پیگیری ادارات
ردیف 5: ردیف
ردیف 6: ردیف | نام پرونده | تاریخ | نتیجه
ردیف 7: 1 | پرونده الف | 1403/03/01 | بررسی شد
ردیف 8: 2 | پرونده ب | 1403/03/05 | در حال پیگیری
ردیف 9: 3 | پرونده ج | 1403/03/10 | نهایی شد
```

**نتیجه:**
- یک کاربرگ ایجاد می‌شود
- 3 فیلد متغیر: "نام پرونده"، "تاریخ"، "نتیجه"
- 3 ردیف داده با `row_index` = 0، 1، 2

### 3. مشاهده فرم در Frontend

API: `GET /api/forms/{id}/form`

Response شامل:
- `fields`: آرایه فیلدها
- `data_rows`: آرایه ردیف‌های داده
- `row_count`: تعداد ردیف‌ها

### 4. ذخیره فرم با چند ردیف

API: `POST /api/forms/{id}/form`

Request:
```json
{
  "data_rows": [
    {
      "row_index": 0,
      "values": {
        "1": "پرونده الف",
        "2": "1403/03/01",
        "3": "بررسی شد"
      }
    },
    {
      "row_index": 1,
      "values": {
        "1": "پرونده ب",
        "2": "1403/03/05",
        "3": "در حال پیگیری"
      }
    }
  ],
  "is_completed": false
}
```

### 5. Export به Excel

API: `GET /api/forms/{id}/export`

خروجی: فایل Excel با فرمت استاندارد و تمام ردیف‌های داده

⚠️ **توجه:** Frontend باید بروز شود تا با API جدید سازگار باشد.

1. **صفحه مشاهده فرم:**

3. **صفحه لیست کاربرگ‌ها:**

## خطاهای احتمالی و راه‌حل

### خطا 1: "Column 'activity_id' not found"
**علت:** Migration اجرا نشده  
**راه‌حل:** `php artisan migrate`

### خطا 2: "Unique constraint violation"
**علت:** تلاش برای ذخیره مقدار تکراری  
**راه‌حل:** بررسی `row_index` در request

**راه‌حل:** ابتدا اقدام را در سیستم ثبت کنید

**علت:** ردیف 6 (عناوین) خالی است
