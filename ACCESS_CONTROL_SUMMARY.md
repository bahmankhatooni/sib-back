# خلاصه کنترل دسترسی - سامانه SIB

## قوانین دسترسی

### 🔑 کاربر Admin (ADMIN)
```
✅ دسترسی کامل به تمام کاربرگ‌ها
✅ مشاهده / ویرایش / حذف همه واحدها
✅ فیلتر بر اساس واحد
✅ دانلود گزارشات کامل
```

### 👤 کاربر عادی (UNIT_RECORDER)
```
✅ دسترسی فقط به کاربرگ‌های واحد خودش
✅ مشاهده / ویرایش / حذف فقط واحد خودش
❌ دسترسی به کاربرگ‌های واحدهای دیگر
❌ فیلتر واحد (مخفی است)
```

## API Endpoints با کنترل دسترسی

### کاربرگ‌ها (Forms)
| Endpoint | Admin | User | توضیحات |
|----------|-------|------|---------|
| `GET /api/forms` | همه | فقط واحد خودش | لیست کاربرگ‌ها |
| `GET /api/forms/{id}` | همه | فقط واحد خودش | جزئیات کاربرگ |
| `POST /api/forms` | همه | واحد خودش | ایجاد کاربرگ |
| `PUT /api/forms/{id}` | همه | فقط واحد خودش | ویرایش کاربرگ |
| `DELETE /api/forms/{id}` | همه | فقط واحد خودش | حذف کاربرگ |
| `GET /api/forms/{id}/export` | همه | فقط واحد خودش | دانلود Excel |

### گزارشات (Reports)
| Endpoint | Admin | User | توضیحات |
|----------|-------|------|---------|
| `GET /api/reports/statistics` | همه واحدها | فقط واحد خودش | آمار کلی |
| `GET /api/reports/list` | همه واحدها | فقط واحد خودش | لیست با فیلتر |
| `GET /api/reports/details/{id}` | همه | فقط واحد خودش | جزئیات کاربرگ |
| `GET /api/reports/export` | همه واحدها | فقط واحد خودش | دانلود Excel |

## پیام‌های خطا

### 403 - ممنوع
```json
{
  "success": false,
  "message": "شما به این کاربرگ دسترسی ندارید"
}
```

**زمان بروز:**
- کاربر عادی تلاش می‌کند به کاربرگ واحد دیگر دسترسی پیدا کند
- کاربر عادی تلاش می‌کند کاربرگ واحد دیگر را ویرایش کند
- کاربر عادی تلاش می‌کند کاربرگ واحد دیگر را حذف کند

### 404 - یافت نشد
```json
{
  "success": false,
  "message": "کاربرگ مورد نظر یافت نشد"
}
```

**زمان بروز:**
- ID کاربرگ نادرست است
- کاربرگ حذف شده است

## مثال‌های عملی

### مثال 1: Admin مشاهده تمام کاربرگ‌ها
```bash
# لاگین
curl -X POST http://localhost:8000/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# دریافت تمام کاربرگ‌ها
curl -X GET http://localhost:8000/api/forms \
  -H "Authorization: Bearer TOKEN"

# نتیجه: تمام کاربرگ‌ها از همه واحدها
```

### مثال 2: کاربر عادی مشاهده فقط واحد خودش
```bash
# لاگین (کاربر واحد 2)
curl -X POST http://localhost:8000/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"user1","password":"pass123"}'

# دریافت کاربرگ‌ها
curl -X GET http://localhost:8000/api/forms \
  -H "Authorization: Bearer TOKEN"

# نتیجه: فقط کاربرگ‌های واحد 2
```

### مثال 3: کاربر عادی تلاش برای دسترسی غیرمجاز
```bash
# کاربر واحد 2 تلاش می‌کند کاربرگ واحد 1 را ببیند
curl -X GET http://localhost:8000/api/forms/5 \
  -H "Authorization: Bearer TOKEN"

# نتیجه: 403 Forbidden
# {
#   "success": false,
#   "message": "شما به این کاربرگ دسترسی ندارید"
# }
```

## سوالات متداول

**Q: چرا کاربر عادی فقط یک کاربرگ می‌بیند؟**  
A: چون فقط یک کاربرگ در واحد او وجود دارد. کاربرگ‌های دیگر متعلق به واحدهای دیگر هستند.

**Q: چطور می‌توانم به عنوان کاربر عادی کاربرگ‌های بیشتری ببینم؟**  
A: باید کاربرگ‌های بیشتری در واحد خودتان ایجاد کنید یا به عنوان Admin لاگین کنید.

**Q: آیا می‌توانم نقش کاربر را تغییر دهم؟**  
A: بله، Admin می‌تواند نقش کاربران را تغییر دهد.

**Q: فیلترهای تاریخ کجا رفتند؟**  
A: حذف شدند چون نیازی به آنها نبود.
