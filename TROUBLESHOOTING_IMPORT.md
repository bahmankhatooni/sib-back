# عیب‌یابی مشکلات Import

## مشکل 1: اقدام در لیست نمایش داده نمی‌شود

### علت‌های احتمالی:

#### 1. فرمت ردیف 3 (اقدام) نادرست است
```
❌ اشتباه: اقدام 30201: نظارت بر محیط زیست
❌ اشتباه: 30201 : نظارت بر محیط زیست  
❌ اشتباه: اقدام ۳۰۲۰۱ : نظارت (اعداد فارسی)

✅ درست: اقدام 30201 : نظارت بر محیط زیست
```

**Regex مورد استفاده:**
```regex
/اقدام\s+(\S+)\s*:\s*(.+)/
```

**توضیح:**
- باید کلمه "اقدام" داشته باشد
- یک یا چند فاصله
- کد اقدام (بدون فاصله)
- صفر یا چند فاصله
- دو نقطه `:`
- صفر یا چند فاصله
- عنوان اقدام

#### 2. اقدام در دیتابیس وجود ندارد

بررسی کنید:
```sql
SELECT id, code, title, program_id 
FROM tasks 
WHERE code = '30201';
```

اگر نتیجه‌ای برنگشت، باید اقدام را ابتدا در سیستم ثبت کنید.

#### 3. اقدام به برنامه اشتباه مربوط است

```sql
SELECT t.id, t.code, t.title, t.program_id, p.code as program_code
FROM tasks t
JOIN programs p ON t.program_id = p.id
WHERE t.code = '30201';
```

مطمئن شوید `program_id` اقدام با `program_id` کاربرگ شما یکسان است.

## مشکل 2: فیلدهای متغیر ذخیره نمی‌شوند

### علت‌های احتمالی:

#### 1. ردیف 6 خالی است یا فقط "ردیف" دارد

```
❌ اشتباه:
ردیف 6: [خالی] [خالی] [خالی]
یا
ردیف 6: ردیف [خالی] [خالی]

✅ درست:
ردیف 6: ردیف | نام | تاریخ | نتیجه
```

#### 2. ردیف 5 را نادیده گرفته‌اید

ساختار صحیح:
```
ردیف 1: هدف X : ...
ردیف 2: برنامه Y : ...
ردیف 3: اقدام Z : ...
ردیف 4: فعالیت W : ...
ردیف 5: ردیف          ← این ردیف باید وجود داشته باشد!
ردیف 6: ردیف | فیلد1 | فیلد2 | فیلد3
ردیف 7: 1 | مقدار1 | مقدار2 | مقدار3
```

**⚠️ نکته مهم:** ردیف 5 فقط شامل کلمه "ردیف" در سلول A5 است.

#### 3. کد به اشتباه ردیف 5 را به عنوان ردیف 6 می‌خواند

اگر ردیف 5 را ندارید، کد ردیف 6 شما را به عنوان ردیف 5 می‌خواند!

**مثال اشتباه:**
```
ردیف 1: هدف...
ردیف 2: برنامه...
ردیف 3: اقدام...
ردیف 4: فعالیت...
ردیف 5: ردیف | فیلد1 | فیلد2    ← کد این را نمی‌خواند!
ردیف 6: 1 | مقدار1 | مقدار2
```

**مثال صحیح:**
```
ردیف 1: هدف...
ردیف 2: برنامه...
ردیف 3: اقدام...
ردیف 4: فعالیت...
ردیف 5: ردیف                    ← فقط "ردیف"
ردیف 6: ردیف | فیلد1 | فیلد2    ← عناوین
ردیف 7: 1 | مقدار1 | مقدار2      ← مقادیر
```

## راه‌حل‌های سریع

### برای مشکل اقدام:

1. **بررسی فرمت:**
   - باز کردن Excel
   - رفتن به ردیف 3 هر شیت
   - مطمئن شدن از فرمت: `اقدام [کد] : [عنوان]`
   - استفاده از اعداد انگلیسی (نه فارسی)

2. **بررسی دیتابیس:**
   ```sql
   -- لیست تمام اقدام‌ها
   SELECT id, code, title FROM tasks;
   
   -- اگر خالی بود، ابتدا اقدام‌ها را ثبت کنید
   ```

3. **Import تست:**
   - ایجاد یک فایل با یک شیت
   - مطمئن شدن از فرمت صحیح
   - Import کردن
   - بررسی نتیجه

### برای مشکل فیلدها:

1. **اضافه کردن ردیف 5:**
   ```
   در Excel:
   - Insert Row بعد از ردیف 4
   - در سلول A5 تایپ کنید: ردیف
   - ذخیره و import
   ```

2. **بررسی ساختار:**
   - شمارش ردیف‌ها: 1، 2، 3، 4، 5، 6، 7
   - مطمئن شدن ردیف 6 عناوین دارد
   - مطمئن شدن ردیف 7+ مقادیر دارند

3. **تست ساده:**
   ```
   فایل تست با ساختار:
   A1: هدف 5 : تست
   A2: برنامه 4 : تست
   A3: اقدام 30201 : تست
   A4: فعالیت 1 : تست
   A5: ردیف
   A6: ردیف | B6: نام | C6: تاریخ
   A7: 1 | B7: آزمایش | C7: 1403/03/01
   ```

## دستورات بررسی دیتابیس

### بررسی آخرین کاربرگ import شده:
```sql
SELECT 
    f.id,
    f.code,
    f.task_id,
    t.code as task_code,
    t.title as task_title,
    (SELECT COUNT(*) FROM form_fields WHERE form_id = f.id) as field_count,
    (SELECT COUNT(*) FROM form_field_values WHERE form_id = f.id) as value_count
FROM forms f
LEFT JOIN tasks t ON f.task_id = t.id
ORDER BY f.created_at DESC
LIMIT 5;
```

### نتیجه مورد انتظار:
```
| id | code | task_id | task_code | task_title | field_count | value_count |
|----|------|---------|-----------|------------|-------------|-------------|
| 10 | 30201-1 | 5 | 30201 | نظارت... | 3 | 9 |
```

اگر:
- `task_id` = NULL → اقدام ذخیره نشده
- `field_count` = 0 → فیلدها ذخیره نشده‌اند
- `value_count` = 0 → مقادیر ذخیره نشده‌اند
