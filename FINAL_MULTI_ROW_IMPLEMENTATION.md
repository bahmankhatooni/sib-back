### 4. بررسی دیتابیس

```sql
-- بررسی ساختار جدول
DESCRIBE form_field_values;

-- باید ستون‌های زیر را داشته باشد:
-- form_id (bigint unsigned)
-- row_index (int)

-- بررسی داده‌ها
SELECT 
    f.code AS form_code,
    ff.field_label,
    ffv.row_index,
    ffv.field_value
FROM form_field_values ffv
JOIN form_fields ff ON ffv.form_field_id = ff.id
JOIN forms f ON ffv.form_id = f.id
WHERE f.id = 1
ORDER BY ffv.row_index, ff.sort_order;

-- نتیجه مورد انتظار: چند ردیف داده با row_index های مختلف
```

## ⚠️ نکات مهم

### 1. داده‌های قدیمی
- Migration فقط ستون‌های جدید را اضافه می‌کند
- داده‌های قدیمی حفظ می‌شوند اما `row_index` آنها 0 است
- اگر داده مهمی دارید، backup بگیرید

### 2. Frontend
Frontend باید بروز شود تا با API جدید سازگار باشد:
- صفحه مشاهده فرم: نمایش جدول چند ردیفه
- صفحه ویرایش فرم: امکان افزودن/حذف/ویرایش ردیف‌ها
- ارسال `data_rows` به جای `fields`

### 3. Performance
برای فایل‌های بزرگ (500+ شیت):
- زمان import: 2-3 دقیقه
- مصرف حافظه: معمولی
- timeout: اگر لازم بود افزایش دهید
